# Senparc.AI AgentKernel Console Sample 使用说明

打开解决方案并编译本控制台项目，或直接 `dotnet run`。

## 配置

1. 使用编辑工具修改 `appsettings.json`（推荐本地开发使用 `appsettings.Development.json` 覆盖敏感信息）
2. 填写 `SenparcAiSetting` 中对应平台的 ApiKey、Endpoint、模型名称
3. 如需 Embedding / RAG，请配置 `ModelName.Embedding` 与 `EmbeddingDimensions`，以及 `VectorDB`
4. 如需 MCP 示例，请配置 `McpSample.Servers`
5. `ToolBindingMode` 默认建议使用 `LocalFunctionProxy`（本地代理为 AIFunction，兼容当前 AgentKernel Chat 路径）
6. 若改为 `HostedServerTool`，`LocalSseUrl` 通常需要配置 `PublicBaseUrl` 或环境变量 `MCP_PUBLIC_BASE_URL` 做公网 URL 映射（会自动保留 `/sse` 路径）

## 运行

### Windows / macOS / Linux

```bash
dotnet run --project Samples/Senparc.AI.Samples.AgentKernelConsoles
```

或在输出目录执行：

```bash
dotnet Senparc.AI.Samples.AgentKernelConsoles.dll
```

## 与 Consoles 示例的区别

- 本示例引用 **Senparc.AI.AgentKernel**，基于 Microsoft Agent Framework
- 对话上下文通过 **AgentSession** 维护
- 当前已覆盖 Chat / Completion / Embedding / RAG / Image / STT / TTS / MCP（LocalFunctionProxy / HostedServerTool）
- 尚未在 AgentKernel 中实现的能力（如 Planner、Plugin）会提示「尚未提供」，可使用 `Senparc.AI.Samples.Consoles`（Kernel 版）
